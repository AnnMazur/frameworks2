Практическое задание №2. Мини веб-служба и конвейер обработки запросов

В работе реализовано приложение с поддержкой модулей, которые можно подключать без изменения ядра.  
Модули загружаются, сортируются с учётом зависимостей и выполняются в правильном порядке.

Запуск
dotnet run

Архитектура

### Ядро (Core)
- Интерфейс `IAppModule`
- Загрузчик модулей `ModuleLoader`
- Проверка:
  - отсутствующих модулей
  - циклических зависимостей
  - 
### Модули
- **HelloModule**
  - базовый модуль
- **TimeModule**
  - выводит текущее время
- **RandomModule**
  - генерирует случайное число
- **LoggingModule**
  - регистрирует логгер
  - выводит итог: сколько модулей отработало
